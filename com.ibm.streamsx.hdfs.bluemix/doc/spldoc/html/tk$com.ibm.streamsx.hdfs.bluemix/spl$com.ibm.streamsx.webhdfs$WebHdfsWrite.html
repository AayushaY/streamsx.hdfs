Composites

composite WebHdfsWrite(input InStream)

Write data to WebHdfs

Formats the tuple with Format, and then sends the blob to WebHdfsWriteBinary. If consistent cut is enabled, the same tuple could be sent to the file multiple times; this could result in an corrupted file if the Format parameter is block. All operators in this composite are fused into one PE.

Parameters

  • url: to access webhdfs. It should start with http://host:port/..webhdfs/v1
  • file: filename to write
  • username: username to use to connector to webhdfs if accessed via a knox gateway
  • password: password to use to connect ot webhdfs if accessed via a knox gateway
  • overwrite: true if hte file should be overwritten if present, false otherwise
  • format: parameter to be passed to Format operator; defaults to line.
  • partition: (optional) colocation id to use.

Input Ports

  • InStream: to write to WebHdfs
